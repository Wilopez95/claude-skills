# Memory Skill

Two modes: **save** and **load**. The memory file lives at `.claude-memory.md` in the current working directory.

---

## /memory save

Summarize the current session into a compact entry and **append** it to `.claude-memory.md`.

### What to capture

Analyze the full conversation and extract:

| Field | What to write |
|---|---|
| `Goal` | The main task or objective of this session (1-2 sentences) |
| `Status` | `completed` / `in-progress` / `blocked` |
| `Decisions` | Key choices made and the reason why (skip obvious ones) |
| `Files` | Files created or modified with a parenthetical note on what changed |
| `Errors` | Problems hit → how they were resolved (or not) |
| `Next` | Concrete remaining steps as a checklist |

Skip anything that wasn't relevant to the session (e.g., no files touched → omit Files).

### Entry format

Keep it tight — each entry should be ~100-200 tokens. Use this template:

```
---
## {ISO date} | {Status}
**Goal:** {one or two sentences}
**Decisions:**
- {decision} ({why})
**Files:** `{path}`({what changed}), `{path}`({what changed})
**Errors:** {error} → {resolution or "unresolved"}
**Next:**
- [ ] {step}
- [ ] {step}
```

### Appending the entry

1. If `.claude-memory.md` doesn't exist, create it with this header:
   ```
   # Claude Session Memory
   <!-- auto-generated by /memory skill -->
   ```
2. Append the new entry at the **bottom** of the file.
3. Confirm: tell the user "Memory saved to `.claude-memory.md`" and echo back the Next steps so they can verify nothing was missed.

### Pruning old entries

If the file has more than **7 entries**, collapse the oldest ones (keeping the 5 most recent in full). Replace each pruned entry with a single summary line:

```
<!-- {date} | {goal one-liner} | {status} -->
```

This keeps the file lean without losing history entirely.

## /memory load

Read `.claude-memory.md` and brief yourself so you can resume the session without the user re-explaining anything.

### Steps

1. Read `.claude-memory.md` from the current working directory.
2. If the file doesn't exist, say: "No memory file found in this directory. Start one with `/memory save` at the end of your next session."
3. Parse all entries. The **most recent** gets full attention; older entries give historical context.

### Briefing format

Output a structured resume message to the user — don't just silently read it. Use this format:

```
## Resuming session — {date of last entry}

**Where we left off:** {goal} — Status: {status}

**Last decisions made:**
- {decision}

**Files in play:** {list}

**What's next:**
- [ ] {step}
- [ ] {step}

{If there are older sessions}: Prior sessions: {one-liner per session, oldest→newest}

Ready to continue. What would you like to tackle first?
```

This makes the context explicit so the user can confirm it's correct before you start working.

---

## Token efficiency notes

- Entries use structured fields, not prose — avoids filler words.
- On load, older sessions are shown as one-liners, not full entries.
- The pruning rule (7 max, collapse to comment) keeps the file from ballooning.
- Prefer bullet points over paragraphs everywhere.
